# 云文档监控助手 - 设计文档

## 一、需求描述

> 文档助手：要爬取阿里云、百度云的官网文档，实时查询是否有内容更新，并让大模型输出总结更新。同时也要能对比百度云跟阿里云的文档进行内容对比，让大模型去输出内容对比。

核心需求：
1. **文档更新监控**：定期检查文档变化，发现更新后让 AI 总结改了什么
2. **跨厂商对比**：对比阿里云和百度云同类产品的功能差异

## 二、实现思路

### 2.1 怎么拿到文档数据

要监控文档更新，第一步是能获取文档内容。

#### 方式一：直接爬 HTML

最直观的想法是请求文档页面，解析 HTML：

```python
import requests
from bs4 import BeautifulSoup

url = "https://help.aliyun.com/zh/ecs/user-guide/security-group-rules"
html = requests.get(url).text
soup = BeautifulSoup(html, "lxml")
content = soup.select_one("article").get_text()
```

**问题：**
- 页面结构可能变化，`article` 选择器哪天就不对了
- 拿不到"最后更新时间"，只能用内容 hash 判断变化
- 有些内容是 JS 动态渲染的，requests 拿不到

#### 方式二：找官方 API

云厂商的文档站点不是纯静态的，背后有数据接口。打开浏览器开发者工具（F12），切到 Network 标签，刷新页面，观察 XHR 请求。

**阿里云文档 API 发现过程：**

在阿里云文档页面抓包，发现两个关键请求：

1. 文档详情 API
```
GET https://help.aliyun.com/help/json/document_detail.json
参数：
  - alias: 文档路径，如 /ecs/user-guide/security-group-rules
  - website: cn
  - language: zh
```

返回数据：
```json
{
  "code": 200,
  "data": {
    "title": "安全组规则",
    "content": "<div>...HTML内容...</div>",
    "lastModifiedTime": 1707123456000,
    "alias": "/ecs/user-guide/security-group-rules"
  }
}
```

关键字段 `lastModifiedTime` 是毫秒时间戳，可以用来判断文档是否更新。

2. 文档目录 API
```
GET https://help.aliyun.com/help/json/menupath.json
参数：
  - alias: 产品下任意文档路径
```

返回整个产品的文档树结构，可以递归提取所有文档的 alias。

**百度云文档 API 发现过程：**

百度云文档站用 Gatsby 框架构建（看页面源码能发现 `___gatsby` 标识）。

Gatsby 的特点是每个页面都有对应的 `page-data.json`，路径规律：
```
页面 URL: https://cloud.baidu.com/doc/VPC/s/qjwvyu0at
数据 URL: https://bce.bdstatic.com/p3m/bce-doc/online/VPC/doc/VPC/s/page-data/qjwvyu0at/page-data.json
```

返回数据：
```json
{
  "result": {
    "data": {
      "markdownRemark": {
        "html": "<div>...文档内容...</div>",
        "fields": {
          "title": "VPC",
          "slug": "qjwvyu0at",
          "date": "2025-01-21"
        }
      }
    }
  }
}
```

百度云的目录获取方式不同 —— 没有单独的目录 API，需要从实际文档页面的 HTML 中解析侧边栏链接：
```python
# 从文档页面 HTML 中提取侧边栏所有链接
pattern = r'href="(/doc/VPC/s/([^"]+))"'
matches = re.findall(pattern, html)
```

#### 结论

| 对比项 | 爬 HTML | 官方 API |
|--------|---------|----------|
| 数据结构 | 不稳定，随页面改版变化 | 稳定，JSON 格式 |
| 更新时间 | 拿不到 | 阿里云有 lastModifiedTime |
| 实现复杂度 | 需要处理各种 HTML 结构 | 直接解析 JSON |
| 维护成本 | 高，页面改了要跟着改 | 低，API 一般不会变 |

**优先用官方 API**，爬 HTML 作为兜底方案。

### 2.2 怎么检测文档是否更新

拿到文档内容后，怎么判断它有没有变化？

**方案一：对比内容 hash**

每次抓取文档，计算内容的 MD5/SHA256，和上次存储的 hash 对比：
```
旧 hash: a1b2c3d4...
新 hash: e5f6g7h8...
不一样 → 有更新
```

问题：文档格式微调（多个空格、换行变化）也会导致 hash 变化，产生大量误报。

**方案二：用官方的更新时间**

阿里云 API 返回 `lastModifiedTime` 字段：
```python
if new_doc.last_modified > old_doc.last_modified:
    return True  # 有更新
```

**结论：优先用时间戳**，没有时间戳时再用 hash 兜底。

### 2.3 怎么让 AI 总结"更新了什么"

检测到文档更新后，要让 AI 总结具体改了什么。

**错误做法：把整篇新文档丢给 AI**
```
prompt: "请总结这篇文档的内容：{新文档全文}"
```
问题：AI 会总结整篇文档讲了什么，而不是"改了什么"。

**正确做法：先生成 diff，再让 AI 分析 diff**
```python
# 1. 生成新旧版本的差异
diff = difflib.unified_diff(旧内容, 新内容)

# 2. 把 diff 给 AI
prompt = f"""
以下是文档《{标题}》的变更内容（diff 格式）：
{diff}

请总结这次更新具体改了什么。
"""
```

这样 AI 的输出才是"变更摘要"而不是"文档摘要"。

### 2.4 怎么对比两个云厂商的产品

用户想知道"阿里云和百度云的安全组有什么不同"。

**难点一：文档结构不同**

阿里云和百度云的文档标题、组织方式完全不同，无法一一对应。

**解决方案：让 AI 从目录中选相关文档**
1. 获取双方产品的完整文档目录
2. 让 AI 从两边目录中分别选出相关文档
3. 抓取选中文档的内容
4. 把双方内容一起给 AI 对比

**难点二：用户要的是"产品功能差异"，不是"文档差异"**

**解决方案：在 prompt 中明确要求**
```
请对比两个云厂商在安全组功能上的**产品能力差异**，
而不是对比文档本身的写法差异。

输出结构：
1. 双方都支持的功能及规格差异
2. 阿里云独有的功能
3. 百度云独有的功能
4. 综合评价
```

### 2.5 怎么知道用户想查哪个产品

用户输入"安全组"，系统怎么知道要去 ECS/BCC 下面找？

**方案一：维护映射表**（需要人工维护，容易遗漏）

**方案二：让 AI 判断**
```
prompt: "用户想查：安全组，请判断它属于阿里云和百度云的哪个产品"
```

**结论：让 AI 做这类判断**，不硬编码。

## 三、整体流程

### 文档更新监控流程
```
定时任务 → 获取文档目录 → 检查更新时间 → 发现更新 → 生成 diff → AI 总结变更
```

### 跨厂商对比流程
```
用户输入 → AI 识别产品 → 获取双方文档 → AI 选相关文档 → 抓取内容 → AI 对比输出
```

```python
def _search_and_get_doc(crawler, query):
    # 1. 让 AI 识别产品
    prompt = f"用户想查：{query}，请判断对应阿里云哪个产品，返回 alias 如 /ecs"
    product = ai.generate(prompt)  # 输出: /ecs
    
    # 2. 获取产品文档目录
    aliases = crawler.discover_product_docs(product)  # 返回 1248 篇文档
    
    # 3. 让 AI 从目录中选择
    toc = "\n".join(f"{i}. {a}" for i, a in enumerate(aliases))
    prompt = f"用户想找：{query}\n目录：\n{toc}\n请选出最相关的文档编号"
    selected = ai.generate(prompt)  # 输出: 42,56,78
    
    # 4. 获取选中文档内容
    doc = crawler.crawl_page(aliases[42])
    return doc
```

#### 变更摘要生成

```python
def summarize_change(old_content, new_content, title):
    # 生成 diff
    diff = difflib.unified_diff(
        old_content.splitlines(),
        new_content.splitlines(),
        fromfile="旧版本",
        tofile="新版本"
    )
    diff_text = "\n".join(diff)
    
    # 让 AI 分析 diff
    prompt = f"""
    以下是文档《{title}》的变更内容：
    {diff_text}
    
    请总结这次更新具体改了什么，包括新增、删除、修改的内容。
    """
    return ai.generate(prompt)
```


## 六、AI 是怎么工作的

整个系统多次调用 AI（通义千问），每次调用都是给 AI 一段文字（prompt），让它返回特定格式的结果。

### 6.1 AI 判断产品

**场景**：用户输入 "安全组"，系统需要知道去哪个产品下找文档。

**做法**：把问题直接问 AI

```
发给 AI 的内容（prompt）：
────────────────────────────────────────
用户想查找的阿里云文档：「安全组」

请判断这个查询对应阿里云哪个产品，返回该产品的 alias 路径前缀。
阿里云产品alias格式为 /产品名（小写），如 /ecs、/vpc、/oss
只输出一个 alias 路径，如 /ecs，不要输出其他内容。
────────────────────────────────────────

AI 返回：
/ecs
```

**为什么这样做**：
- AI 有云产品的知识，知道安全组是 ECS 的功能
- 不需要维护映射表，AI 自己判断
- 用户说 "防火墙" "访问控制规则" 也能识别出是安全组

### 6.2 AI 选择文档

**场景**：拿到了 ECS 产品的 1248 篇文档目录，需要找出和 "安全组" 相关的。

**做法**：把目录列表给 AI，让它选编号

```
发给 AI 的内容（prompt）：
────────────────────────────────────────
用户想查找的内容：「安全组」

以下是该产品的文档目录（编号. alias路径）：
0. /ecs/product-overview/what-is-ecs
1. /ecs/product-overview/benefits
2. /ecs/product-overview/scenarios
...
42. /ecs/user-guide/security-group-rules
43. /ecs/user-guide/basic-security-groups-and-advanced-security-groups
44. /ecs/user-guide/security-groups-for-different-use-cases
...
1247. /ecs/api-reference/xxx

请从目录中选出与用户查询最相关的1-3篇文档。
只输出选中文档的编号，用逗号分隔，不要输出其他内容。
例如：5,12,23
────────────────────────────────────────

AI 返回：
42,43,44
```

**为什么这样做**：
- 文档目录的 alias 路径包含关键词（security-group）
- AI 能理解语义，即使路径里没有 "安全组" 两个字也能匹配
- 比关键词匹配更智能

### 6.3 AI 总结变更

**场景**：检测到文档更新了，需要告诉用户具体改了什么。

**做法**：先生成新旧版本的 diff，再让 AI 分析

```python
diff = difflib.unified_diff(旧内容.splitlines(), 新内容.splitlines())
```

```
1. 企业级安全组的实例数量限制从 65535 调整为无限制
2. 新增了安全组规则优先级设置功能

影响范围：使用企业级安全组的用户将获得更大的扩展空间...
```

**为什么这样做**：
- 直接给整篇文档，AI 会总结文档内容，不是变更内容
- 给 diff，AI 只关注变化的部分
- 输出的是 "改了什么"，不是 "文档讲了什么"

### 6.4 AI 对比产品

**场景**：用户想知道阿里云和百度云安全组的功能差异。

**做法**：把双方文档内容都给 AI，明确要求对比产品功能

```
发给 AI 的内容（prompt）：
────────────────────────────────────────
以下是阿里云和百度云关于安全组的文档内容。
请对比两个云厂商在该功能上的**产品能力差异**，
而不是对比文档本身的写法差异。

## 阿里云文档内容
【安全组规则】
安全组是一种虚拟防火墙...
规则数量上限：1000条
支持协议：TCP、UDP、ICMP...

【普通安全组和企业级安全组】
普通安全组：组内默认互通...
企业级安全组：支持更大规模...

## 百度云文档内容
【安全组概述】
安全组是百度智能云提供的...
规则数量上限：100条（50入+50出）
支持优先级：1-1000...

请按以下结构输出：
1. 双方都支持的功能及规格差异
2. 阿里云独有的功能
3. 百度云独有的功能
4. 综合评价
────────────────────────────────────────

AI 返回：
## 1. 双方都支持的功能

| 功能 | 阿里云 | 百度云 |
|------|--------|--------|
| 规则数量上限 | 1000条 | 100条 |
| 优先级范围 | 1-100 | 1-1000 |
...

## 2. 阿里云独有功能
- 组内连通策略
- 有状态连接（会话保持910秒）
...

## 3. 百度云独有功能
- 规则拒绝策略
- 安全组复制功能
...
```

**关键点**：
- prompt 里明确说 "产品能力差异，不是文档写法差异"
- 给出输出结构，让 AI 按格式返回
- 双方文档内容都给，AI 自己对比

### 6.5 总结

| AI 任务 | 输入 | 输出 | 关键技巧 |
|---------|------|------|----------|
| 判断产品 | 用户关键词 | 产品 alias | 让 AI 只输出路径，不要解释 |
| 选择文档 | 文档目录列表 | 文档编号 | 让 AI 只输出数字，用逗号分隔 |
| 总结变更 | diff 内容 | 变更摘要 | 给 diff 不给全文，聚焦变化 |
| 对比产品 | 双方文档内容 | 功能差异表 | 明确要求对比功能，不是文档 |

**核心思路**：把 AI 当成一个能理解自然语言的函数，输入是 prompt，输出是特定格式的结果。prompt 写得越清晰，输出越准确。
